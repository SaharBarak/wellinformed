Chapter 02 — The opposite shape

wellinformed is the opposite shape.

Every instance is a peer. Every peer runs the same code. Your graph lives on your CPU, your peers' graphs live on theirs, the protocol lets them share rooms without anyone brokering the exchange.

Every node is a peer.

No wellinformed server. No directory service. No signaling provider. Just ed25519 identities and multiaddrs.

Every peer runs the same code.

MIT-licensed, 21 MCP tools, 396 tests. The binary you run is the binary your peers run. No client/server asymmetry.

Research compounds across peers.

When a peer you trust has already investigated the topic, their embeddings and synthesis flow straight into your session. You inherit the finished work.

The network layer opens only when you say.

Three system rooms ship always-on. Every other room is negotiable via the interactive share picker, one toggle at a time.

Chapter 03 — Identity you own

Your identity is a W3C
did:key —
not a customer record.

Generated locally at first boot. Ed25519 keypair, 32-byte pubkey, multicodec-prefixed per the W3C spec. The user DID is long-lived and survives device changes — rotate a compromised device, keep the identity, never email support.

01
Device authorization chain.

User signs (device_id, device_pub, authorized_at). Devices are revocable without losing the user DID — rotate a compromised laptop, keep the identity.

02
Signed envelopes at the wire.

Every outbound node — memory entry, oracle answer, room share — can be wrapped and verified offline. Three Ed25519 checks, sub-2 ms, zero DID-resolver calls.

03
Canonical JSON, zero new crypto deps.

Key-sorted deterministic encoding — byte-identical across Node versions and architectures. Pure node:crypto PKCS8/SPKI paths, 641 lines of auditable domain code.
